## Core Design Principles

### 1. Intentional, Not Default
Every visual choice must have a reason. Never use a Tailwind default "because it's there." If `rounded-lg` is used, it's because the component calls for it — not because AI defaults to it. If a gradient appears, it serves the brand — not because gradients are "modern."

### 2. Typographic Hierarchy is King
- Use dramatic size contrasts (3x+ between heading and body, not 1.5x)
- Mix font weights aggressively: 300 for elegance, 700-900 for impact
- Line-height matters: tighter for headings (1.1-1.2), relaxed for body (1.6-1.7)
- Letter-spacing: slightly negative for large headings, slightly positive for small caps/labels

### 3. Whitespace as a Design Element
- Use generous whitespace intentionally — it signals premium quality
- Asymmetric spacing creates visual interest (don't center everything)
- Section padding should vary: 120px-160px for major sections, not uniform 64px everywhere

### 4. Color with Restraint
- Max 2-3 colors per section. The accent gradient is a highlight, not a default
- Use the warm gold accent sparingly — it's special when rare
- Dark sections (`#0f172a`) for emphasis, white for breathing room
- Subtle background textures (dot grids, noise, subtle radials) beat flat white

### 5. Motion with Purpose
- Every animation must answer "what does this communicate?"
- Stagger children with 50-80ms delays for cascading reveals
- Use scroll-triggered animations via GSAP for marketing pages
- Respect `prefers-reduced-motion` ALWAYS

### 6. Depth and Layering
- Use multiple background layers (radial gradients, subtle patterns, blurred orbs)
- Cards should feel elevated: combine `shadow-lg` with subtle border
- Glassmorphism used sparingly, not as default card style
- Z-axis thinking: overlay elements, offset grids, overlapping sections

## The Anti-Vibe-Code Checklist

Before submitting ANY frontend code, verify:

- [ ] NOT using Inter as the only font (see fonts-and-typography.md for alternatives)
- [ ] NO purple/indigo gradients anywhere
- [ ] NO uniform `rounded-lg` on everything — vary border-radius by component
- [ ] NO three-icon-boxes-in-a-row as a features section
- [ ] NO centered-hero-with-single-CTA as the only layout pattern
- [ ] NO `shadow-sm` on everything — shadows should vary by elevation
- [ ] NO cookie-cutter repeating card sections
- [ ] NO generic placeholder testimonials
- [ ] Headlines are NOT all the same size
- [ ] Layout has visual variety — not section after section of centered content
- [ ] At least one section breaks the grid (offset, overlap, asymmetry)
- [ ] Animations are staggered and purposeful, not identical fade-ups on everything
- [ ] Background is NOT flat white everywhere — use textures, gradients, patterns
- [ ] Buttons have distinct hierarchy (primary gradient, secondary outlined, ghost)
- [ ] Color contrast passes WCAG AA (4.5:1 text, 3:1 UI elements)

## Component Design Standards

### Buttons
```
Primary:    bg-gradient-accent text-brand-primary font-semibold px-6 py-3 rounded-xl shadow-md hover:shadow-lg transition-all
Secondary:  border border-brand-border text-brand-primary font-medium px-5 py-2.5 rounded-lg hover:bg-brand-surface transition-colors
Ghost:      text-brand-muted font-medium hover:text-brand-primary transition-colors underline-offset-4 hover:underline
```

### Cards
- Use varying border-radius: `rounded-2xl` for feature cards, `rounded-xl` for smaller cards
- Combine subtle border + shadow — never just one
- Hover: translate-y-[-2px] + shadow increase, NOT scale transform
- Content padding: `p-6` minimum, `p-8` for feature cards

### Forms
- Labels always visible (never placeholder-only)
- Input focus: `ring-2 ring-brand-accent-from/50` (warm gold ring, not default blue)
- Error states: red text below input + red ring, never just color change
- Group related fields, use fieldsets semantically
- Submit buttons full-width on mobile, auto-width on desktop

### Navigation
- Desktop: Clean horizontal nav, max 5-6 items, CTA button stands out
- Mobile: Full-screen overlay (not tiny hamburger dropdown)
- Active state: subtle bottom border or background, not bold text alone
- Sticky with `backdrop-blur-lg` and border-bottom on scroll

### Data Tables (Dashboard)
- Sticky header, alternating row tints
- Sort indicators in header cells
- Row hover: subtle warm tint, not gray
- Pagination: proper page numbers, not just prev/next
- Empty state: illustration + CTA, not "No data found"

## Layout Patterns That Don't Look AI-Generated

### Hero Sections
INSTEAD of centered text + CTA + placeholder image, try:
- **Split hero**: Text left, product screenshot/demo right with perspective tilt
- **Full-bleed hero**: Dark background, text overlay, subtle animated particles
- **Asymmetric hero**: Large heading offset left, smaller description right, CTA below
- **Video hero**: Looping video background with text overlay and scroll CTA

### Feature Sections
INSTEAD of 3 icon boxes in a row, try:
- **Bento grid**: Mixed-size cards in a masonry/grid layout
- **Alternating rows**: Feature + visual alternating left/right with GSAP scroll animations
- **Interactive tabs**: Feature categories that reveal different content
- **Horizontal scroll carousel**: GSAP-powered horizontal feature scroll
- **Timeline**: Vertical timeline showing feature progression

### Social Proof
INSTEAD of generic quote cards, try:
- **Logo marquee**: Scrolling client logos (already have marquee CSS)
- **Metric counters**: Animated numbers (leads generated, response time, etc.)
- **Case study cards**: Mini case studies with before/after metrics
- **Video testimonials**: Embedded video with transcript

### Pricing
INSTEAD of 3 columns with feature lists, try:
- **Comparison table**: Full feature matrix with row highlights
- **Interactive calculator**: Slider-based pricing
- **Single featured plan**: Highlight one plan, collapse others
- **Toggle annual/monthly**: With savings badge

## Accessibility Requirements

These are non-negotiable:
- Semantic HTML: `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`
- All images: meaningful `alt` text or `alt=""` for decorative
- Focus indicators: visible, high-contrast, consistent
- Keyboard navigation: Tab order logical, Enter/Space activate, Escape closes
- ARIA: Only when semantic HTML is insufficient
- Color: Never convey information by color alone
- Motion: Always wrap in `prefers-reduced-motion` media query
- Touch targets: 44x44px minimum on mobile
- Heading hierarchy: One `h1` per page, no skipped levels
